# Drizzle migrations

This directory holds Drizzle Kit migration artifacts (`migrations/` subfolder, generated by `pnpm db:generate`) and the RLS pattern documentation (`rls/`).

## Workflow

```bash
# 1. Edit schema files under src/database/schema/
# 2. Generate a SQL migration from the schema diff
pnpm db:generate         # creates apps/api/drizzle/migrations/NNNN_<name>.sql

# 3. Apply migrations to the configured DATABASE_URL
pnpm db:migrate

# Dev shortcut (Phase 0/1 only — DO NOT use in prod): push schema directly without a migration file
pnpm db:push

# Visual DB browser
pnpm db:studio
```

## Phase 0 status

The Phase 0 schema covers `accounts`, `users`, and `audit_log` — the minimum the JWT guard and audit trail need. The first migration generates after Phase 0 finishing:

```bash
cd apps/api
pnpm db:generate -- --name=init_phase_0
pnpm db:migrate
```

Real per-tenant tables (`engagements`, `milestones`, `tickets`, `documents`, `invoices`) land in Phase 2 along with their RLS policies — see `rls/README.md`.

## Conventions

- One concept per schema file under `src/database/schema/` (e.g. `engagements.ts`, `milestones.ts`).
- Every per-tenant table has an `account_id uuid NOT NULL REFERENCES accounts(id)` column. No exceptions.
- Every table has `created_at` and `updated_at` timestamps with timezone.
- Soft deletes are opt-in per table — when used, add `deleted_at` with a partial index.
- Enum types live with the table that owns them.
